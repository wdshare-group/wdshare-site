  • 浏览器默认样式的不一致
  • 反项目的默认样式
  • 项目中设计规范的通用样式

CSS reset 起始

据玉伯的叙述，第一个CSS reset是Tantek写的undohtml.css。

粗暴的CSS reset

*{ margin:0; padding:0; }

错杀一千，不放过一个。偷懒时候常用。

滥用的CSS reset

摘自淘宝用的CSS Reset:
blockquote, body, button, dd, dl, dt, fieldset, form, h1, h2, h3, h4, h5, h6, hr, input, legend, li, ol, p, pre, td, textarea, th, ul{margin:0;padding:0}
  • body只要margin:0;就over了
  • p标签只需要margin:0;或margin:1em 0;
  • dt在所有浏览器中表现都是一致的

滥用的CSS Reset 后果

  • 不当的破坏默认样式。比如粗暴的 *{ margin:0; padding:0; }
  • 浏览器的可用性样式。a标签的outline
  • 浏览器针对组件的一些样式。比如input标签的对其属性和间距。
  • 某些特定情况下，独立性很高的第三方插件。如富文本编辑器都要对css reset全部覆写。
  • 不太重要的性能问题

合理简洁的CSS reset

body, dl, dd, p, blockquote, ul, ol, pre, form, h1, h2, h3, h4, h5, h6{margin:0;}
ul, ol, th, td{padding:0;}

normalize.css

nicolas gallagher

浏览器层面通用的方案，只覆写不一致的user agent样式，保留有用的user agent样式，同时进行一些 bug 的修复。
  • normalize v3（ie8+）
  • normalize v1.3.1(兼容ie6 7 8)

覆写反项目的样式

不同人眼中，反项目的样式是不同的。normalize仅仅只做了保持一致。
normalize 可能对项目合理的
p标签 margin: 1em 0; margin:0;
ul,ol margin: 0 0 0 40px; margin:40px; list-style:none;

添加通用样式

  • body{margin:0; font:normal 12px/1.5 '微软雅黑';}
  • a{color:#34538b;},a:hover{underline:none;}
  • img{border:0;vertical-align:middle}
  • table{border-collapse:collapse;border-spacing:0}
